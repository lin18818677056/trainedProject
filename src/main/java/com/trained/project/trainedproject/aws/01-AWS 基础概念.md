## 1.1 什么是 AWS？

### 1.1.1 AWS 简介

**Amazon Web Services (AWS)** 是全球领先的云计算服务平台，由亚马逊公司提供。

简单来说，AWS 让你可以通过互联网租用计算资源，而不需要自己购买和维护物理服务器。

### 1.1.2 类比理解

想象一下：

**传统方式（自建机房）**
```
你需要开餐厅 → 买地 → 建房 → 装修 → 买厨具 → 雇厨师 → 开始营业
耗时：6 个月
投入：几百万
风险：生意不好就亏本
```

**使用 AWS（云计算）**
```
你需要开餐厅 → 租用现成的厨房 → 按需付费 → 开始营业
耗时：几分钟
投入：按使用量付费
风险：生意不好可以随时退租
```

## 1.2 云计算的三种服务模式

### 1.2.1 IaaS - 基础设施即服务

**Infrastructure as a Service**

- **提供**: 虚拟机、存储、网络等基础设施
- **你管理**: 操作系统、中间件、运行时、应用程序和数据
- **AWS 示例**: EC2（虚拟服务器）

```
┌─────────────────────────┐
│     你管理的部分         │
│  应用 + 数据 + 运行时    │
│  + 中间件 + 操作系统     │
├─────────────────────────┤
│   AWS 管理的基础设施     │
│   (虚拟化、服务器等)     │
└─────────────────────────┘
```

**适用场景**: 
- 需要完全控制环境
- 迁移现有应用到云端
- 开发和测试环境

### 1.2.2 PaaS - 平台即服务

**Platform as a Service**

- **提供**: 开发平台和工具
- **你管理**: 应用程序和数据
- **AWS 示例**: Elastic Beanstalk

```
┌─────────────────────────┐
│     你管理的部分         │
│      应用 + 数据         │
├─────────────────────────┤
│   AWS 管理的平台         │
│  运行时 + 中间件 + OS   │
│  + 虚拟化 + 服务器       │
└─────────────────────────┘
```

**适用场景**:
- 应用程序开发
- 快速部署
- 团队协作开发

### 1.2.3 SaaS - 软件即服务

**Software as a Service**

- **提供**: 完整的应用程序
- **你管理**: 仅配置和使用
- **示例**: Gmail, Salesforce, Office 365

```
┌─────────────────────────┐
│     你管理的部分         │
│        仅配置使用        │
├─────────────────────────┤
│   AWS 管理的完整应用     │
│  应用 + 数据 + 所有底层  │
└─────────────────────────┘
```

## 1.3 AWS 的核心优势

### 1.3.1 成本优化

| 传统 IT | AWS |
|---------|-----|
| 大量前期投资 | 按需付费 |
| 固定成本 | 可变成本 |
| 资源闲置浪费 | 只为使用的付费 |

**例子**:
```
传统方式:
买服务器花费：¥50,000
使用率：30%
浪费：¥35,000

AWS 方式:
按小时付费：¥0.5/小时
月使用 100 小时：¥50
节省：99%+
```

### 1.3.2 弹性扩展

**垂直扩展 (Scale Up)**
- 增加单个服务器的配置（CPU、内存）
- 适合数据库等场景

**水平扩展 (Scale Out)**
- 增加服务器数量
- 适合 Web 应用

```
流量低时：2 台服务器
    ↓
流量高峰：自动增加到 20 台
    ↓
流量恢复：自动减少到 2 台
```

### 1.3.3 全球覆盖

AWS 在全球拥有广泛的基础设施：

- **区域 (Regions)**: 地理上的数据中心集群
- **可用区 (Availability Zones)**: 区域内的独立数据中心
- **边缘站点 (Edge Locations)**: 内容分发节点

**亚太地区包括**:
- 北京区域
- 宁夏区域
- 香港区域
- 东京区域
- 首尔区域
- 新加坡区域
- 悉尼区域
- 孟买区域

### 1.3.4 安全性

AWS 提供企业级安全：
- 数据加密
- 访问控制
- 合规认证
- DDoS 防护

## 1.4 AWS 核心服务概览

### 1.4.1 计算服务

#### EC2 (Elastic Compute Cloud)
- **功能**: 虚拟服务器
- **用途**: 运行应用程序
- **计费**: 按秒/小时计费

```
类似：租用一台电脑
你可以：安装系统、运行软件
费用：按使用时间
```

#### Lambda
- **功能**: 无服务器计算
- **用途**: 运行代码无需管理服务器
- **计费**: 按执行次数和时间

```
类似：叫外卖
你不需要：厨房、厨师
只需要：点餐、付费
```

### 1.4.2 存储服务

#### S3 (Simple Storage Service)
- **功能**: 对象存储
- **用途**: 存储文件、图片、视频
- **特点**: 高可靠、易扩展

```
类似：云盘/U 盘
可以存：照片、文档、视频
特点：随时随地访问
```

#### EBS (Elastic Block Store)
- **功能**: 块存储
- **用途**: EC2 的硬盘
- **特点**: 高性能

```
类似：电脑的硬盘
必须：挂载到 EC2 使用
特点：速度快
```

### 1.4.3 数据库服务

#### RDS (Relational Database Service)
- **功能**: 关系型数据库
- **支持**: MySQL, PostgreSQL, Oracle, SQL Server
- **管理**: 自动备份、补丁更新

```
类似：托管的数据库服务
你负责：创建表、写 SQL
AWS 负责：维护、备份
```

#### DynamoDB
- **功能**: NoSQL 数据库
- **特点**: 快速、可扩展
- **用途**: 游戏、物联网

### 1.4.4 网络服务

#### VPC (Virtual Private Cloud)
- **功能**: 隔离的虚拟网络
- **用途**: 控制网络访问
- **安全**: 防火墙规则

```
类似：在你家周围建围墙
可以控制：谁能进、谁不能进
```

#### Route 53
- **功能**: DNS 服务
- **用途**: 域名解析
- **特点**: 高可用

### 1.4.5 其他重要服务

| 服务 | 功能 | 用途 |
|------|------|------|
| IAM | 身份和访问管理 | 控制用户权限 |
| CloudWatch | 监控服务 | 查看资源使用情况 |
| SNS/SQS | 消息通知/队列 | 应用间通信 |
| API Gateway | API 管理 | 创建和管理 API |

## 1.5 AWS 定价模型

### 1.5.1 计费方式

#### 按需付费 (On-Demand)
- 按实际使用量付费
- 无承诺
- 灵活性最高

#### 预留实例 (Reserved Instances)
- 承诺使用 1 年或 3 年
- 折扣可达 72%
- 适合稳定负载

#### Spot 实例
- 竞价购买闲置容量
- 折扣可达 90%
- 可能被中断

### 1.5.2 免费套餐 (Free Tier)

新账户享受 12 个月免费套餐：

**EC2**:
- 750 小时/月 t2.micro 或 t3.micro
- 相当于 1 台服务器 24 小时运行

**S3**:
- 5GB 标准存储
- 20,000 次 GET 请求
- 2,000 次 PUT 请求

**RDS**:
- 750 小时/月 db.t2.micro
- 20GB 存储

**Lambda**:
- 每月 100 万次请求
- 400,000 GB-秒计算时间

### 1.5.3 成本估算示例

**场景**: 小型网站

```
EC2 t3.small (2 vCPU, 2GB 内存):
  ¥0.068/小时 × 24 小时 × 30 天 = ¥48.96/月

S3 10GB 存储:
  ¥0.078/GB/月 × 10GB = ¥0.78/月

RDS db.t3.micro:
  ¥0.052/小时 × 24 × 30 = ¥37.44/月

总计：约 ¥87/月
```

## 1.6 AWS 管理控制台

### 1.6.1 登录方式

1. **Web 控制台**: https://console.aws.amazon.com
2. **CLI 命令行**: 适合自动化
3. **SDK**: 编程访问
4. **移动 App**: AWS Console App

### 1.6.2 控制台界面

主要区域：
- **顶部导航栏**: 服务搜索、账户信息
- **左侧菜单**: 服务分类
- **主区域**: 服务内容
- **Dashboard**: 服务概览

### 1.6.3 常用功能

#### 服务搜索
```
在顶部搜索框输入服务名称
如：EC2, S3, RDS
快速访问服务
```

#### 区域选择
```
右上角选择区域
推荐：亚太区域（离中国近）
注意：不同区域价格可能不同
```

#### 账单查看
```
搜索 "Billing"
查看当前费用
设置预算告警
```

## 1.8 常见问题解答

### Q1: AWS 难学吗？
**A**: 只要有耐心和实践，任何人都能学会。关键是多动手操作。

### Q2: 学习 AWS 需要编程基础吗？
**A**: 基础使用不需要，但深入使用建议学习一些编程知识。

### Q3: 学生有优惠吗？
**A**: 有！AWS Educate 为学生提供额外资源和积分。

### Q4: 中文支持怎么样？
**A**: AWS 控制台支持中文，文档也有中文版。

### Q5: 在中国能用 AWS 吗？
**A**: 可以，AWS 在中国有北京和宁夏区域。
