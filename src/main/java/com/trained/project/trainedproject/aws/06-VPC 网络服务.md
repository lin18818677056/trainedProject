## 6.1 VPC 基础概念

### 6.1.1 什么是 VPC？

**VPC = Virtual Private Cloud（虚拟私有云）**

简单理解：
```
VPC = 你在 AWS 云中的私人网络

在这个网络里:
  ✓ 你决定 IP 地址范围
  ✓ 你划分子网
  ✓ 你控制路由规则
  ✓ 你配置防火墙
  ✓ 完全隔离其他用户
```

### 6.1.2 类比理解

| 传统网络 | VPC |
|----------|-----|
| 公司机房 | VPC |
| 交换机 | 子网 |
| 路由器 | 路由表 |
| 防火墙 | 安全组/NACL |
| 网线 | 网络接口 |

### 6.1.3 核心组件

```
VPC (虚拟私有云)
  ├── 子网 (Subnet)
  │     ├── 公有子网
  │     └── 私有子网
  ├── 互联网网关 (IGW)
  ├── NAT 网关
  ├── 路由表
  ├── 安全组
  └── 网络 ACL
```

## 6.2 VPC 核心组件详解

### 6.2.1 VPC 和 CIDR

**CIDR 表示法**:
```
CIDR = Classless Inter-Domain Routing
格式：IP 地址/前缀长度

例子:
  10.0.0.0/16
  172.31.0.0/16
  192.168.0.0/16
```

**推荐 VPC 网段**:
```
✓ 10.0.0.0/16 (65,536 个 IP)
✓ 172.31.0.0/16 (65,536 个 IP)
✓ 192.168.0.0/16 (65,536 个 IP)
```

**子网划分示例**:
```
VPC: 10.0.0.0/16

划分为:
  - 10.0.1.0/24 (公有子网 A)
  - 10.0.2.0/24 (私有子网 A)
  - 10.0.3.0/24 (公有子网 B)
  - 10.0.4.0/24 (私有子网 B)
```

### 6.2.2 子网 (Subnet)

**什么是子网？**
```
子网 = VPC 内的更小网段
位于单个可用区
包含一组 IP 地址
```

**子网类型**:

#### 公有子网
```
特点:
  ✓ 有路由到互联网网关
  ✓ 可以有公网 IP
  ✓ 可从互联网访问
  
用途:
  - Web 服务器
  - 负载均衡器
  - NAT 网关
```

#### 私有子网
```
特点:
  ✓ 无直接互联网路由
  ✓ 只有内网 IP
  ✓ 不可直接从外网访问
  
用途:
  - 数据库
  - 应用服务器
  - 缓存服务
```

### 6.2.3 互联网网关 (IGW)

**作用**:
```
✓ VPC ↔ 互联网的桥梁
✓ 允许出入互联网流量
✓ 每个 VPC 最多一个 IGW
```

**架构图**:
```
互联网
   ↓
互联网网关 (IGW)
   ↓
VPC
   ├── 公有子网
   └── 私有子网
```

### 6.2.4 NAT 网关

**NAT = Network Address Translation**

**作用**:
```
让私有子网的资源:
  ✓ 可以访问互联网
  ✗ 不能被互联网访问
  
单向访问:
  私有子网 → 互联网 ✓
  互联网 → 私有子网 ✗
```

**使用场景**:
```
✓ 私有子网 EC2 下载更新
✓ 访问外部 API
✓ 上传数据到 S3（如果不用 VPC 端点）
```

**部署位置**:
```
必须放在公有子网
需要 Elastic IP
高可用需多 AZ 部署
```

### 6.2.5 路由表

**什么是路由表？**
```
路由表 = 网络流量的方向指引

定义:
  - 数据包往哪里去
  - 通过哪个网关
  - 目标网络是什么
```

**默认路由**:
```
0.0.0.0/0 → 互联网网关
  表示：所有未知流量都走互联网网关

10.0.0.0/16 → local
  表示：VPC 内部流量本地路由
```

### 6.2.6 安全组 vs NACL

| 特性 | 安全组 | NACL |
|------|--------|------|
| 层级 | 实例级 | 子网级 |
| 状态 | 有状态 | 无状态 |
| 规则 | 只允许 | 允许/拒绝 |
| 处理顺序 | 所有规则 | 按序号 |
| 默认 | 拒绝所有入站 | 允许所有 |

**安全组**:
```
✓ 作用于 EC2 实例
✓ 有状态（返回流量自动允许）
✓ 只能设置允许规则
✓ 第一道防线
```

**NACL**:
```
✓ 作用于整个子网
✓ 无状态（需分别配置进出）
✓ 可设置允许/拒绝
✓ 第二道防线
```

## 6.3 创建 VPC 和网络架构

### 6.3.1 使用 VPC 向导（推荐新手）

#### 步骤 1: 进入 VPC 控制台
```
1. AWS 控制台搜索 "VPC"
2. 点击进入 VPC 服务
```

#### 步骤 2: 选择模板
```
VPC 仪表板 → 创建 VPC

选择模板:
  ☑ 具有公共和私有子网的 VPC
  
这个模板包括:
  ✓ VPC
  ✓ 公有子网（2 个）
  ✓ 私有子网（2 个）
  ✓ 互联网网关
  ✓ NAT 网关
  ✓ 路由表
```

#### 步骤 3: 配置参数

```
IPv4 CIDR 块：10.0.0.0/16

子网设置:
  公有子网 1:
    - 可用区：us-east-1a
    - CIDR: 10.0.1.0/24
    
  公有子网 2:
    - 可用区：us-east-1b
    - CIDR: 10.0.2.0/24
    
  私有子网 1:
    - 可用区：us-east-1a
    - CIDR: 10.0.10.0/24
    
  私有子网 2:
    - 可用区：us-east-1b
    - CIDR: 10.0.11.0/24

NAT 网关:
  ☑ 在一个可用区中（学习用）
  ○ 在每个可用区中（生产用）

标签:
  Name: My-First-VPC
```

#### 步骤 4: 预览和创建
```
检查所有配置
点击 "创建 VPC"
等待创建完成（约 2-5 分钟）
```

### 6.3.2 手动创建 VPC（深入学习）

#### 步骤 1: 创建 VPC
```
1. VPC 控制台 → 您的 VPC
2. 创建 VPC

名称：My-Custom-VPC
IPv4 CIDR: 10.0.0.0/16
IPv6: 不启用
十进制：默认
```

#### 步骤 2: 创建子网

**公有子网 A**:
```
名称：Public-Subnet-A
VPC: My-Custom-VPC
可用区：us-east-1a
IPv4 CIDR: 10.0.1.0/24
```

**私有子网 A**:
```
名称：Private-Subnet-A
VPC: My-Custom-VPC
可用区：us-east-1a
IPv4 CIDR: 10.0.10.0/24
```

#### 步骤 3: 创建互联网网关
```
1. 左侧菜单：互联网网关
2. 创建互联网网关

名称：My-IGW
VPC: 先不附加

3. 创建后附加到 VPC
   操作 → 附加到 VPC
   选择：My-Custom-VPC
```

#### 步骤 4: 创建 NAT 网关
```
1. 左侧菜单：NAT 网关
2. 创建 NAT 网关

名称：My-NAT-Gateway
子网：Public-Subnet-A（公有子网）
弹性 IP: 分配新 EIP
```

**注意**: NAT 网关每小时收费，学习完记得删除

#### 步骤 5: 创建路由表

**公有路由表**:
```
1. 路由表 → 创建路由表

名称：Public-Route-Table
VPC: My-Custom-VPC

2. 编辑路由
   添加路由:
     - 目标：0.0.0.0/0
     - 目标：互联网网关
   
3. 保存
```

**私有路由表**:
```
1. 创建路由表

名称：Private-Route-Table
VPC: My-Custom-VPC

2. 编辑路由
   添加路由:
     - 目标：0.0.0.0/0
     - 目标：NAT 网关
   
3. 保存
```

#### 步骤 6: 关联子网

**公有子网关联**:
```
1. 选择 Public-Route-Table
2. 子网关联 → 编辑
3. 勾选 Public-Subnet-A
4. 保存
```

**私有子网关联**:
```
1. 选择 Private-Route-Table
2. 子网关联 → 编辑
3. 勾选 Private-Subnet-A
4. 保存
```

## 6.4 EC2 实例网络配置

### 6.4.1 在 VPC 中启动 EC2

#### 公有子网实例
```
1. EC2 控制台 → 启动实例

2. 网络设置:
   VPC: My-Custom-VPC
   子网：Public-Subnet-A
   自动分配公有 IP: 启用
   
3. 安全组:
   允许 SSH (22) 从你的 IP
   允许 HTTP (80) 从任意 IP
   
4. 启动
```

**结果**:
```
✓ 实例有公网 IP
✓ 可以从互联网访问
✓ 可以访问互联网
```

#### 私有子网实例
```
1. 启动实例

2. 网络设置:
   VPC: My-Custom-VPC
   子网：Private-Subnet-A
   自动分配公有 IP: 禁用
   
3. 安全组:
   根据需要配置
   
4. 启动
```

**结果**:
```
✓ 实例只有内网 IP
✓ 不能从互联网直接访问
✓ 可以通过 NAT 访问互联网
✓ 可通过跳板机访问
```

### 6.4.2 弹性 IP (EIP)

**什么是弹性 IP？**
```
✓ 静态的公网 IP 地址
✓ 可以绑定到实例
✓ 实例终止后保留
✓ 免费（绑定运行实例时）
✓ 收费（未绑定时）
```

**分配弹性 IP**:
```
1. VPC 控制台 → 弹性 IP
2. 分配地址
3. 分配
```

**关联弹性 IP**:
```
1. 选择 EIP
2. 操作 → 关联地址
3. 选择实例
4. 关联
```

**解关联**:
```
1. 选择 EIP
2. 操作 → 解除关联
3. 确认
```

## 6.5  bastion host（跳板机）

### 6.5.1 什么是跳板机？

```
跳板机 = 访问私有子网的桥梁

架构:
互联网
   ↓
公有子网：Bastion Host (有公网 IP)
   ↓ SSH
私有子网：数据库/应用 (无公网 IP)
```

### 6.5.2 创建跳板机

**步骤 1: 启动公有子网 EC2**
```
AMI: Amazon Linux 2
实例类型：t2.micro
子网：Public-Subnet-A
自动分配公有 IP: 启用

安全组:
  - SSH (22) 从你的 IP
```

**步骤 2: 配置私有子网实例**
```
安全组:
  - SSH (22) 仅从跳板机安全组
```

**步骤 3: 连接到跳板机**
```bash
ssh -i key.pem ec2-user@跳板机公网 IP
```

**步骤 4: 从跳板机连接私有实例**
```bash
# 在跳板机上执行
ssh -i key.pem ec2-user@私有实例内网 IP
```

### 6.5.3 SSH 端口转发（单命令访问）

```bash
ssh -i key.pem \
  -J ec2-user@跳板机公网 IP \
  ec2-user@私有实例内网 IP
```

**解释**:
```
-J: 跳板选项
先连接跳板机
再转发到私有实例
```

## 6.6 VPC 对等连接

### 6.6.1 什么是 VPC 对等？

```
VPC Peering = 两个 VPC 之间的直接连接

特点:
  ✓ 低延迟
  ✓ 不走互联网
  ✓ 私有连接
  ✓ 同区域或跨区域
```

### 6.6.2 创建对等连接

**前提**:
```
✓ 两个 VPC 的 CIDR 不重叠
✓ 例如：10.0.0.0/16 和 10.1.0.0/16
```

**步骤**:

**VPC A 配置**:
```
1. VPC 控制台 → 对等连接
2. 创建对等连接

名称：Peering-VPCA-to-VPCB
VPC (请求方): VPC-A
区域：同一区域
VPC (接受方): VPC-B

3. 创建
```

**VPC B 配置**:
```
1. 在对等连接列表找到请求
2. 操作 → 接受请求
3. 确认接受
```

**配置路由**:

**VPC A 路由表**:
```
添加路由:
  - 目标：VPC-B 的 CIDR (10.1.0.0/16)
  - 目标：对等连接 ID (pcx-xxx)
```

**VPC B 路由表**:
```
添加路由:
  - 目标：VPC-A 的 CIDR (10.0.0.0/16)
  - 目标：对等连接 ID (pcx-xxx)
```

**验证**:
```bash
# 在 VPC-A 的实例 ping VPC-B 的实例
ping 10.1.0.10
```

## 6.7 VPC 流日志

### 6.7.1 什么是流日志？

```
流日志 = 记录网络流量信息

记录内容:
  - 源 IP 和目标 IP
  - 端口号
  - 协议
  - 数据包数量
  - 接受/拒绝
```

### 6.7.2 创建流日志

```
1. VPC 控制台
2. 选择 VPC/子网/网络接口
3. 流日志标签
4. 创建流日志

配置:
  筛选条件：全部
  目标：发送到 CloudWatch Logs
  日志组：新建或选择
  IAM 角色：自动创建
  
5. 创建
```

### 6.7.3 查看流日志

```
1. CloudWatch → 日志 Insights
2. 选择日志组
3. 查询分析

示例查询:
fields @timestamp, srcAddr, dstAddr, action
| filter action = 'REJECT'
| sort @timestamp desc
```

## 6.9 故障排查

### 问题 1: 无法连接 EC2

**现象**:
```
SSH 连接超时
```

**排查步骤**:
```
1. 检查实例状态：是否运行中
2. 检查安全组：是否开放 22 端口
3. 检查路由表：是否有到 IGW 的路由
4. 检查 NACL: 是否允许 SSH 流量
5. 检查公网 IP: 是否分配了公网 IP
```

### 问题 2: 私有子网无法访问互联网

**检查清单**:
```
□ NAT 网关是否在公有子网
□ NAT 网关是否有 EIP
□ 私有子网路由表是否指向 NAT
□ 公有子网路由表是否指向 IGW
□ 安全组是否允许出站
```

### 问题 3: VPC 对等不通

**排查**:
```
1. 检查对等连接状态：是否 active
2. 检查双方路由表：是否有对方路由
3. 检查安全组：是否允许对方 IP
4. 检查 CIDR: 是否重叠
5. 测试连通性：ping/traceroute
```

### 问题 4: NACL 阻止流量

**调试方法**:
```
1. 查看 NACL 规则
2. 检查入站和出站规则
3. 注意规则序号（从小到大）
4. 确保有 ALLOW ALL 规则
5. 临时禁用 NACL 测试
```

## 6.13 常见问题 FAQ

### Q1: VPC 免费吗？
**A**: VPC 本身免费，但其中的资源（NAT、EC2 等）收费。

### Q2: 可以修改 VPC 的 CIDR 吗？
**A**: 可以添加额外 CIDR，但不能修改主 CIDR。

### Q3: NAT 网关和 NAT 实例有什么区别？
**A**: 
- NAT 网关：托管服务，高可用，自动扩展
- NAT 实例：自己管理 EC2，便宜但需维护

### Q4: 如何迁移现有 EC2 到 VPC？
**A**: 创建 AMI，然后在 VPC 中启动新实例。

### Q5: VPC 之间流量收费吗？
**A**: 通过对等连接免费（同区域）。跨区域收费。

### Q6: 一个 VPC 可以有多少个子网？
**A**: 理论上 200 个，实际根据需要规划。

### Q7: 如何测试网络连通性？
**A**: 使用 ping, traceroute, telnet 等工具。
