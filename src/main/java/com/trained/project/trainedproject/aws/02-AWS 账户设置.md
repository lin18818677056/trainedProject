## 2.1 注册前的准备

### 2.1.1 必需材料

请提前准备好以下物品：

```
✅ 电子邮箱（推荐 Gmail、Outlook 等国际邮箱）
✅ 信用卡或借记卡（Visa、MasterCard 等）
✅ 手机号码（用于接收验证码）
✅ 身份证件（可能需要身份验证）
```

### 2.1.2 银行卡要求

**支持的卡类型**:
- ✅ Visa 信用卡/借记卡
- ✅ MasterCard 信用卡/借记卡
- ✅ American Express
- ⚠️ 银联卡：部分支持，建议准备 Visa/MasterCard

**重要提示**:
```
⚠️ 卡必须有足够余额验证（约$1 预授权）
⚠️ 预授权会在几天内退还
⚠️ 确保开通了国际支付功能
```

### 2.1.3 邮箱选择

**推荐**:
- Gmail (gmail.com)
- Outlook (outlook.com)
- Yahoo (yahoo.com)

**不推荐**:
- QQ 邮箱（可能收不到验证邮件）
- 163 邮箱（可能收不到验证邮件）

- 最少 8 个字符

- 允许特殊符号：!@#$%^&* 等

```
❌ password123 (无大写字母)
❌ ALLCAPS123 (无小写字母)
❌ 12345678 (无字母)
```

## 2.4 安全设置（极其重要！）

### 2.4.1 启用 MFA 多因素认证

**什么是 MFA?**
```
MFA = Multi-Factor Authentication
类似：登录密码 + 手机验证码
双重保护，即使密码泄露也安全
```

**启用步骤**:

1. 点击右上角账户名称
2. 选择"安全凭证"
3. 找到"多因素认证 (MFA)"
4. 点击"分配 MFA 设备"

**选择设备类型**:

```
选项 1: 虚拟 MFA 设备（推荐）
  - 使用手机 App
  - 免费
  - 方便

选项 2: 硬件 MFA 设备
  - 物理密钥
  - 需要购买
  - 更安全
```

**使用虚拟 MFA 设备**:

```
步骤 1: 下载认证器 App
  - iOS: Google Authenticator / Microsoft Authenticator
  - Android: Google Authenticator / Microsoft Authenticator
  
步骤 2: 扫描二维码
  - 打开 App
  - 扫描屏幕上的二维码
  
步骤 3: 输入验证码
  - App 会生成 6 位数字
  - 连续输入两个验证码
  - 间隔 30 秒
  
步骤 4: 完成
  - 保存恢复代码（重要！）
  - 存放在安全地方
```

**保存的恢复代码示例**:
```
XXXX-XXXX-XXXX-XXXX
XXXX-XXXX-XXXX-XXXX
```
⚠️ **警告**: 丢失恢复代码可能导致账户无法恢复！

### 2.4.2 创建访问密钥（Access Keys）

**什么是访问密钥？**
```
Access Key ID: AKIAIOSFODNN7EXAMPLE
Secret Access Key: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY

用于：CLI、SDK 编程访问
重要性：等同密码，必须保密
```

**创建步骤**:

1. 进入"安全凭证"页面
2. 找到"访问密钥"
3. 点击"创建访问密钥"
4. 选择使用场景（推荐"其他"）
5. 勾选确认框
6. 点击"下一步"
7. **立即下载 CSV 文件**
8. 点击"完成"

**⚠️ 重要提醒**:

```
✅ 立即历史 CSV 文件
✅ 将 CSV 文件保存在安全地方
✅ Secret Key 只显示一次，丢失需重新创建
✅ 不要分享给任何人
✅ 不要上传到代码仓库
```

**正确保存方式**:
```
❌ 错误：
  - 放在桌面
  - 发到微信文件传输助手
  - 上传到 GitHub
  
✅ 正确：
  - 加密的 U 盘
  - 密码管理器（如 1Password、LastPass）
  - 离线存储
```

```

```

**详细步骤**:

```

```

#### 步骤 3: 设置预算详情

```
预算名称：Monthly-Budget-10USD
预算类型：周期性预算
周期：每月

金额设置:
  定期预算金额：10 USD
  开始月份：当前月份
  
汇率：使用 AWS 默认汇率
```

#### 步骤 4: 设置警报阈值

```
添加警报 1:
  阈值：80%
  实际或预测：实际

添加警报 2:
  阈值：100%
  实际或预测：预测
```

## 2.6 创建第一个 IAM 用户

### 2.6.1 为什么创建 IAM 用户？

**根账户 vs IAM 用户**:

| 根账户 | IAM 用户 |
|--------|----------|
| 拥有所有权限 | 可限制权限 |
| 用于账户管理 | 用于日常操作 |
| 应该保护起来 | 可以正常使用 |
| 只有一个 | 可以创建多个 |

**最佳实践**:
```
✅ 根账户：仅用于账户管理（改密码、看账单）
✅ IAM 用户：日常使用（创建 EC2、S3 等）
✅ 最小权限：只给必要的权限
```

### 2.6.2 创建管理员 IAM 用户

**详细步骤**:

#### 步骤 1: 进入 IAM 控制台
```
1. 搜索 "IAM"
2. 点击进入 IAM 服务
```

#### 步骤 2: 开始创建用户
```
1. 左侧菜单：用户
2. 点击"添加用户"
```

#### 步骤 3: 设置用户详情

```
用户名：admin-falak
访问类型：
  ☑ 编程访问（Access key）
  ☑ 控制台访问（Console password）
  
控制台密码:
  ○ 自动生成密码
  ● 自定义密码：YourPassword123!
  
用户必须在新密码创建后更改它：☐ 不勾选
```

#### 步骤 4: 设置权限

**方式 1: 直接附加策略（简单）**
```
选择：附加现有策略直接
  
搜索并选择:
  ☑ AdministratorAccess
  
说明：这个策略有全部权限，仅用于学习测试
```

**方式 2: 添加到组（推荐）**
```
1. 创建用户组：AdminGroup
2. 附加策略：AdministratorAccess
3. 将用户加入组
```

#### 步骤 5: 添加标签（可选）
```
键：Department
值：IT

键：Project
值：Learning
```

#### 步骤 6: 审核并创建

```
检查所有设置是否正确
点击：创建用户
```

#### 步骤 7: 下载凭证

```
⬇️ 下载 .csv 文件
  - 包含 Access Key ID
  - 包含 Secret Access Key
  - 包含控制台登录链接
  
⚠️ 重要：
  - 立即历史 CSV 文件
  - 安全保存
  - Secret Key 只显示一次
```

### 2.6.3 使用 IAM 用户登录

**登录步骤**:

1. 打开 IAM 控制台

## 2.7 配置 CLI（可选但推荐）

### 2.7.1 安装 AWS CLI

**Windows 系统**:

```powershell
# 方法 1: MSI 安装包
# 1. 下载安装程序
https://awscli.amazonaws.com/AWSCLIV2.msi

# 2. 双击运行安装
# 3. 按照向导完成安装

# 方法 2: 使用 Chocolatey
choco install awscli
```

**验证安装**:
```powershell
aws --version
```

输出应该是：
```
aws-cli/2.x.x Python/3.x.x Windows/10 exe/x86_64
```

### 2.7.2 配置凭证

**步骤**:

1. 打开命令提示符或 PowerShell
2. 输入配置命令：

```powershell
aws configure
```

3. 按提示输入：

```
AWS Access Key ID [None]: AKIAIOSFODNN7EXAMPLE
AWS Secret Access Key [None]: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
Default region name [None]: ap-southeast-1
Default output format [None]: json
```

**参数说明**:
- **Access Key ID**: 从下载的 CSV 文件获取
- **Secret Access Key**: 从 CSV 文件获取
- **Region**: 选择的区域代码
  - `ap-southeast-1` (新加坡)
  - `ap-northeast-1` (东京)
  - `us-east-1` (美国北部)
- **Output format**: 输出格式（json/text/table）

### 2.7.3 测试配置

```powershell
# 测试 S3 命令
aws s3 ls

# 测试 EC2 命令
aws ec2 describe-instances

# 如果返回结果，说明配置成功！
```

## 2.9 故障排查

### 问题 1: 收不到验证邮件

**解决方案**:
```
1. 检查垃圾邮件文件夹
2. 换个邮箱（推荐 Gmail）
3. 等待 15 分钟
4. 重新发送验证邮件
```

### 问题 2: 信用卡验证失败

**解决方案**:
```
1. 确认卡支持国际支付
2. 联系银行开通境外交易
3. 确保有足够余额（$1）
4. 换一张信用卡
```

### 问题 3: 无法登录控制台

**解决方案**:
```
1. 检查邮箱和密码
2. 重置密码
3. 清除浏览器缓存
4. 尝试无痕模式
5. 联系 AWS 支持
```

### 问题 4: Access Key 失效

**解决方案**:
```
1. 在 IAM 控制台删除旧密钥
2. 创建新的访问密钥
3. 更新本地配置
4. 更新应用程序配置
```

## 2.12 常见问题 FAQ

### Q1: 注册时显示"账户已存在"怎么办？
**A**: 说明你用这个邮箱注册过，尝试找回密码或直接登录。

### Q2: 学生有额外优惠吗？
