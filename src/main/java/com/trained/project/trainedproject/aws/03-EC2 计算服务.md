## 3.1 EC2 基础概念

### 3.1.1 什么是 EC2？

**EC2 = Elastic Compute Cloud（弹性云计算）**

简单理解：
```
EC2 = 云端的虚拟电脑

你可以：
  ✓ 选择操作系统（Windows/Linux）
  ✓ 选择配置（CPU、内存大小）
  ✓ 安装软件和运行应用
  ✓ 按使用时间付费
```

### 3.1.2 类比理解

| 传统方式 | EC2 方式 |
|----------|----------|
| 买物理服务器 | 租用虚拟服务器 |
| 一次性投入数万 | 每小时几毛钱 |
| 维护硬件设备 | AWS 负责硬件维护 |
| 资源固定 | 随时调整配置 |

### 3.1.3 使用场景

**适合用 EC2 的情况**:
```
✅ 搭建网站或 Web 应用
✅ 开发和测试环境
✅ 运行企业应用
✅ 数据处理和分析
✅ 游戏服务器
✅ 机器学习训练
```

**不适合用 EC2 的情况**:
```
❌ 静态网站（用 S3 更便宜）
❌ 短期批处理任务（考虑 Lambda）
❌ 需要大量数据库操作（考虑 RDS）
```

## 3.2 EC2 核心组件

### 3.2.1 AMI（Amazon Machine Image）

**什么是 AMI？**
```
AMI = 预配置的模板
包含：操作系统 + 软件 + 配置

类似：电脑的 Ghost 系统盘
```

**AMI 类型**:

#### 1. Amazon Linux 2（推荐新手）
```
优点:
  ✓ 免费（不收系统授权费）
  ✓ 针对 AWS 优化
  ✓ 安全稳定
  
适用：学习、一般应用
```

#### 2. Ubuntu Server
```
优点:
  ✓ 社区活跃
  ✓ 文档丰富
  ✓ 软件包多
  
适用：Web 开发、容器
```

#### 3. Windows Server
```
优点:
  ✓ 图形界面
  ✓ .NET 应用支持
  
注意：需要额外授权费用
```

#### 4. CentOS/RHEL
```
优点:
  ✓ 企业级稳定
  ✓ 长期支持
  
注意：RHEL 需要授权费
```

### 3.2.2 实例类型（Instance Type）

**命名规则**:
```
系列 + 代际 + 大小

例如：t3.micro
  t = 系列
  3 = 代际
  micro = 大小
```

**常见实例系列**:

#### 通用型 (General Purpose) - 推荐新手
```
T3/T2 系列：突发性能，适合学习
  t3.micro:   2 vCPU, 1GB 内存
  t3.small:   2 vCPU, 2GB 内存
  t3.medium:  2 vCPU, 4GB 内存

M5 系列：平衡性能
  m5.large:   2 vCPU, 8GB 内存
  m5.xlarge:  4 vCPU, 16GB 内存
```

#### 计算优化型 (Compute Optimized)
```
C5 系列：高 CPU 性能
  c5.large:   2 vCPU, 4GB 内存
  c5.xlarge:  4 vCPU, 8GB 内存
  
适用：批量处理、游戏服务器
```

#### 内存优化型 (Memory Optimized)
```
R5 系列：大内存
  r5.large:   2 vCPU, 16GB 内存
  r5.xlarge:  4 vCPU, 32GB 内存
  
适用：数据库、大数据分析
```

### 3.2.3 存储选项

#### EBS (Elastic Block Store)
```
类似：电脑的硬盘

特点:
  ✓ 持久化存储
  ✓ 高性能
  ✓ 可以挂载到 EC2
  
类型:
  - GP3/GP2: 通用型（推荐）
  - IO1/IO2: 高性能型
  - ST1: 吞吐优化
  - SC1: 冷存储
```

#### 实例存储 (Instance Store)
```
类似：临时 U 盘

特点:
  ✓ 超高性能
  ✗ 实例终止后数据丢失
  
适用：缓存、临时数据
```

### 3.2.4 安全组 (Security Group)

**什么是安全组？**
```
安全组 = 虚拟防火墙

作用：控制进出 EC2 的流量

规则:
  - 入站规则：允许哪些流量进入
  - 出站规则：允许哪些流量出去
```

**重要特性**:
```
✓ 有状态：允许进入的流量自动允许返回
✓ 默认拒绝所有入站
✓ 默认允许所有出站
✓ 只能设置允许规则，不能设置拒绝规则
```

## 3.3 创建第一台 EC2 实例（详细步骤）

### 3.3.1 准备工作

**确认事项**:
```
□ AWS 账户已激活
□ 已完成 MFA 设置
□ 已创建 IAM 用户
□ 知道要选择的区域
```

### 3.3.2 步骤一：进入 EC2 控制台

```
1. 登录 AWS 控制台
2. 在搜索框输入 "EC2"
3. 点击 "EC2" 服务
4. 进入 EC2 Dashboard
```

### 3.3.3 步骤二：启动实例

```
1. 点击橙色按钮 "启动实例"
2. 或者点击左侧 "实例" → "启动实例"
```

### 3.3.4 步骤三：配置实例（7 个步骤）

#### **第 1 步：选择 AMI**

```
快速启动 AMI 推荐:

☑ Amazon Linux 2 AMI (HVM) - Kernel 5.10
   SSD Volume Type
   
   架构：64 位 (x86)
   根设备类型：EBS
   虚拟化类型：HVM
   
   点击 "选择"
```

其他选项：
- Ubuntu Server 22.04 LTS
- Red Hat Enterprise Linux
- SUSE Linux
- Windows Server

#### **第 2 步：选择实例类型**

```
新手推荐:

☑ t2.micro 或 t3.micro
   符合条件：1 台实例 750 小时/月 免费
  
   配置：
   - vCPU: 1
   - 内存：1 GB
   
   点击 "下一步：配置实例详细信息"
```

**实例类型对比**:

| 类型 | vCPU | 内存 | 价格/小时 | 适用场景 |
|------|------|------|-----------|----------|
| t2/t3.micro | 1 | 1GB | $0.0116 | 学习、测试 |
| t2/t3.small | 1 | 2GB | $0.0232 | 小型网站 |
| t2/t3.medium | 2 | 4GB | $0.0464 | 中型应用 |

#### **第 3 步：配置实例详细信息**

```
数量：1 （默认）

购买选项：按需实例

生命周期：无（不勾选 Spot）

网络：选择默认 VPC
  - 如果没有，保持默认即可

子网：任意（默认）

公有 IP: 启用
  ☑ 分配公有 IP
  
  说明：这样才能从外网访问

IAM 实例配置文件：无
  - 暂时不需要

关闭保护：不勾选
  - 防止误删除

监控：启用基本监控
  - 免费套餐包含

详细监控：不启用
  - 会产生额外费用

点击 "下一步：添加存储"
```

#### **第 4 步：添加存储**

```
新卷配置:

根卷 (/dev/xvda):
  类型：gp3 (通用型 SSD)
  大小：8 GiB (最小值)
  
  免费套餐：30GB 以内免费
  
加密：不加密
  - 学习用途不需要

快照 ID: 无

点击 "下一步：标记实例"
```

**存储建议**:
```
学习用途：8-10 GB 足够
生产环境：根据应用需求
最大限制：16 TiB
```

#### **第 5 步：添加标签**

```
标签用于管理和识别

推荐添加:

标签 1:
  键：Name
  值：MyFirstServer
  
标签 2:
  键：Environment
  值：Learning
  
标签 3:
  键：Project
  值：AWS-Study

点击 "下一步：配置安全组"
```

#### **第 6 步：配置安全组（关键！）**

**选项 A: 使用现有安全组**
```
如果有 default 安全组，可以选择
但通常默认规则不够
```

**选项 B: 创建新安全组（推荐）**

```
☑ 创建新安全组

安全组名称：sg-web-server
描述：允许 HTTP 和 SSH 访问

规则配置:

【入站规则】

规则 1:
  类型：SSH
  协议：TCP
  端口范围：22
  来源：我的 IP
    - 自动填写你的当前 IP
    - 或选择 0.0.0.0/0（不推荐）
    
规则 2:
  类型：HTTP
  协议：TCP
  端口范围：80
  来源：0.0.0.0/0
    - 允许所有人访问 Web
    
规则 3 (可选):
  类型：HTTPS
  协议：TCP
  端口范围：443
  来源：0.0.0.0/0

【出站规则】
  默认全部允许
  保持默认即可

点击 "审查并启动"
```

**⚠️ 安全警告**:
```
❌ 不要随意开放 0.0.0.0/0 的 SSH（22 端口）
✅ SSH 只开放给自己的 IP
✅ Web 端口（80/443）可以开放给所有人
```

#### **第 7 步：审查并启动**

```
检查所有配置:

✓ AMI: Amazon Linux 2
✓ 实例类型：t2.micro
✓ 网络：默认 VPC
✓ 存储：8GB gp3
✓ 安全组：sg-web-server

确认无误后:
  点击 "启动"
```

### 3.3.5 步骤四：创建密钥对

**什么是密钥对？**
```
密钥对 = 登录密码的替代

私钥 (.pem 文件): 你保存
公钥：存储在 AWS

类似：银行的 U 盾
```

**创建步骤**:

```
1. 选择密钥对选项

选项 1: 创建新密钥对（推荐）
  密钥对名称：my-first-key
  点击下载.pem 文件
  
选项 2: 使用已有密钥对
  从下拉列表选择

2. ⬇️ 下载密钥对文件
   - 文件名：my-first-key.pem
   - 保存到安全位置

3. ⚠️ 重要提醒:
   ✓ 这是唯一一次下载机会
   ✓ 丢失无法恢复
   ✓ 必须妥善保管
```

**保存密钥的最佳实践**:

```
❌ 错误做法:
  - 放在桌面
  - 发到微信
  - 上传 GitHub
  - 放在 Downloads 文件夹

✅ 正确做法:
  - 创建专用目录 ~/.ssh/
  - 设置正确的权限
  - 备份到加密存储
  - 使用密码管理器
```

**Windows 系统保存方法**:

```powershell
# 1. 创建.ssh 目录
New-Item -ItemType Directory -Path "$HOME\.ssh" -Force

# 2. 移动密钥文件
Move-Item -Path "$HOME\Downloads\my-first-key.pem" -Destination "$HOME\.ssh\"

# 3. 设置权限（PowerShell 7+）
icacls "$HOME\.ssh\my-first-key.pem" /reset
icacls "$HOME\.ssh\my-first-key.pem" /grant:r "$($env:USERNAME):(R)" /inheritance:r

# 4. 验证权限
Get-Acl "$HOME\.ssh\my-first-key.pem" | Format-List
```

**Linux/Mac 系统保存方法**:

```bash
# 1. 移动到.ssh 目录
mv ~/Downloads/my-first-key.pem ~/.ssh/

# 2. 设置权限
chmod 400 ~/.ssh/my-first-key.pem

# 3. 验证
ls -l ~/.ssh/my-first-key.pem
```

### 3.3.6 步骤五：启动实例

```
1. 点击 "启动实例" 按钮
2. 看到成功消息："您已成功启动实例"
3. 点击实例 ID 查看详情
```

**查看实例状态**:

```
实例状态：正在运行
状态检查：通过
系统状态：通过
```

**等待时间**: 
```
通常 30 秒 - 2 分钟
实例会从"正在启动"变为"正在运行"
```

## 3.4 连接 EC2 实例

### 3.4.1 连接方式概览

| 方式 | 适用系统 | 难度 | 推荐度 |
|------|---------|------|--------|
| EC2 Instance Connect | 所有 | ⭐ | ⭐⭐⭐⭐⭐ |
| SSH (Linux/Mac) | Linux/Mac | ⭐⭐ | ⭐⭐⭐⭐ |
| PuTTY (Windows) | Windows | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| Session Manager | 所有 | ⭐⭐ | ⭐⭐⭐⭐⭐ |

### 3.4.2 方式一：EC2 Instance Connect（最简单）

**前提条件**:
```
✓ 实例是 Amazon Linux 2 或 Ubuntu
✓ 实例有公网 IP
✓ 安全组允许 22 端口
```

**连接步骤**:

```
1. 进入 EC2 控制台
2. 左侧菜单：实例
3. 选中你的实例
4. 点击 "连接" 按钮

5. 选择 "EC2 Instance Connect"
6. 点击 "连接"

7. 浏览器会打开新标签
8. 出现终端窗口即连接成功
```

**优点**:
```
✓ 无需下载密钥
✓ 无需安装软件
✓ 浏览器直接使用
✓ 自动处理认证
```

### 3.4.3 方式二：使用 SSH（Linux/Mac）

**前提条件**:
```
✓ 已下载.pem 密钥文件
✓ 已设置正确权限
✓ 终端工具可用
```

**连接命令**:

```bash
# 基本格式
ssh -i "密钥路径" ec2-user@实例公网 IP

# 实际例子
ssh -i "~/.ssh/my-first-key.pem" ec2-user@54.123.45.67
```

**详细步骤**:

```bash
# 1. 找到实例的公网 IP
# 在 EC2 控制台查看"公有 IPv4 地址"

# 2. 打开终端
# Mac: 打开 Terminal
# Linux: 打开 Terminal

# 3. 输入 SSH 命令
ssh -i "~/.ssh/my-first-key.pem" ec2-user@54.123.45.67

# 4. 首次连接会提示确认
The authenticity of host '54.123.45.67' can't be established.
Are you sure you want to continue connecting (yes/no)?
输入：yes

# 5. 连接成功
Last login: Mon Jan 15 10:30:00 2024 from xxx.xxx.xxx.xxx

       __|  __|_  )
       _|  (     /   Amazon Linux 2 AMI
      ___|\___|___|

https://aws.amazon.com/amazon-linux-2/
```

**不同 AMI 的默认用户名**:

| AMI 类型 | 用户名 |
|----------|--------|
| Amazon Linux 2 | ec2-user |
| Ubuntu | ubuntu |
| CentOS | centos |
| Debian | admin |
| RHEL | ec2-user |
| SUSE | ec2-user |
| root (不推荐) | root |

### 3.4.4 方式三：使用 PuTTY（Windows）

**步骤一：下载 PuTTY**

```
1. 访问：https://www.putty.org/
2. 下载 PuTTY 安装包
3. 安装 PuTTY
```

**步骤二：转换密钥格式**

PuTTY 需要.ppk 格式，需转换.pem 文件：

```
1. 打开 PuTTYgen
   (开始菜单 → PuTTY → PuTTYgen)

2. 点击 "Load"

3. 选择你的.pem 文件
   注意：选择"所有文件 (*.*)"

4. 点击 "Save private key"
   保存为.ppk 格式

5. 输入密钥短语（可选但推荐）
   例如：MySecurePass123!

6. 保存.ppk 文件
```

**步骤三：配置 PuTTY 连接**

```
1. 打开 PuTTY

2. 配置会话:
   Host Name: ec2-user@54.123.45.67
   Port: 22
   Connection type: SSH

3. 加载密钥:
   左侧菜单：Connection → SSH → Auth
   Private key file for authentication: 
     浏览选择刚才保存的.ppk 文件

4. 保存会话（可选）:
   回到 Session 类别
   Saved Sessions: MyEC2Server
   点击 "Save"

5. 点击 "Open"
```

**步骤四：连接**

```
1. 首次连接会提示确认
   Are you sure you want to continue connecting?
   点击 "Yes"

2. 输入用户名（如果没在 Host Name 中指定）
   login as: ec2-user

3. 连接成功
   ec2-user@ip-10-0-0-1:~$
```

### 3.4.5 方式四：Session Manager（最安全）

**优点**:
```
✓ 无需公网 IP
✓ 无需开放 22 端口
✓ 无需密钥文件
✓ 完整的审计日志
✓ 浏览器直接访问
```

**前提条件**:
```
✓ 实例安装了 SSM Agent
✓ IAM 角色配置了权限
✓ VPC 端点配置（私有子网）
```

**连接步骤**:

```
1. EC2 控制台选择实例

2. 点击 "连接"

3. 选择 "Session Manager"

4. 点击 "连接"

5. 终端打开
```

**配置要求较高，新手可先用其他方式**。

## 3.5 基本 Linux 命令

连接成功后，你需要一些基本的 Linux 命令。

### 3.5.1 文件和目录操作

```bash
# 查看当前目录
pwd
# 输出：/home/ec2-user

# 列出文件
ls
ls -la  # 显示详细信息

# 创建目录
mkdir myapp

# 进入目录
cd myapp

# 返回上级目录
cd ..

# 回到家目录
cd ~

# 创建空文件
touch index.html

# 复制文件
cp file1.txt file2.txt

# 移动/重命名文件
mv oldname.txt newname.txt

# 删除文件
rm file.txt

# 删除目录
rm -rf directory
```

### 3.5.2 文件编辑（nano 编辑器）

```bash
# 用 nano 打开文件
nano index.html

# 编辑内容
<!DOCTYPE html>
<html>
<head><title>My Site</title></head>
<body><h1>Hello AWS!</h1></body>
</html>

# 保存退出:
Ctrl + O → Enter → Ctrl + X
```

### 3.5.3 系统信息

```bash
# 查看系统信息
uname -a

# 查看磁盘空间
df -h

# 查看内存
free -h

# 查看 CPU
lscpu

# 查看操作系统版本
cat /etc/os-release
```

### 3.5.4 软件安装（Amazon Linux 2）

```bash
# 更新软件包列表
sudo yum update -y

# 安装 Apache
sudo yum install httpd -y

# 安装 Nginx
sudo yum install nginx -y

# 安装 MySQL
sudo yum install mysql -y

# 安装 PHP
sudo yum install php -y

# 安装 Python 3
sudo yum install python3 -y

# 安装 Git
sudo yum install git -y

# 安装 Node.js
curl -sL https://rpm.nodesource.com/setup_14.x | sudo bash -
sudo yum install nodejs -y
```

### 3.5.5 服务管理

```bash
# 启动服务
sudo systemctl start httpd

# 停止服务
sudo systemctl stop httpd

# 重启服务
sudo systemctl restart httpd

# 查看状态
sudo systemctl status httpd

# 开机自启
sudo systemctl enable httpd

# 禁止开机自启
sudo systemctl disable httpd
```

### 3.5.6 权限管理

```bash
# 更改文件所有者
sudo chown ec2-user myfile.txt

# 更改文件权限
chmod 755 script.sh

# 以 root 身份执行命令
sudo yum update

# 切换到 root 用户
sudo -i
# 退出：exit
```

## 3.6 实战：部署 Web 应用

### 3.6.1 安装 Apache Web 服务器

```bash
# 1. 更新系统
sudo yum update -y

# 2. 安装 Apache
sudo yum install httpd -y

# 3. 启动 Apache
sudo systemctl start httpd

# 4. 设置开机自启
sudo systemctl enable httpd

# 5. 查看状态
sudo systemctl status httpd
```

### 3.6.2 配置安全组

确保你的安全组允许 HTTP 流量：

```
入站规则:
  类型：HTTP
  端口：80
  来源：0.0.0.0/0
```

### 3.6.3 测试默认页面

## 3.7 实例状态管理

### 3.7.1 实例状态

| 状态 | 说明 | 计费 |
|------|------|------|
| Pending | 正在启动 | 不计费 |
| Running | 运行中 | 计费 |
| Stopping | 正在停止 | 计费 |
| Stopped | 已停止 | 仅存储费 |
| Starting | 正在启动 | 计费 |
| Terminating | 正在终止 | 计费到终止前 |
| Terminated | 已终止 | 不计费 |

### 3.7.2 停止 vs 终止

**停止 (Stop)**:
```
✓ 实例关机
✓ 数据保留（EBS 卷）
✓ 仍收取存储费
✓ 可以再次启动
✓ 公网 IP 会改变
```

**终止 (Terminate)**:
```
✓ 实例被删除
✓ EBS 卷默认删除
✓ 不再收费
✓ 无法恢复
✓ 数据永久丢失
```

### 3.7.3 操作实例

**在 EC2 控制台**:

```
1. 选择实例
2. 点击 "实例状态" 按钮

选项:
  - 启动实例
  - 停止实例
  - 重启实例
  - 终止实例
```

**使用 CLI**:

```bash
# 停止实例
aws ec2 stop-instances --instance-ids i-1234567890abcdef0

# 启动实例
aws ec2 start-instances --instance-ids i-1234567890abcdef0

# 重启实例
aws ec2 reboot-instances --instance-ids i-1234567890abcdef0

# 终止实例
aws ec2 terminate-instances --instance-ids i-1234567890abcdef0
```

## 3.10 故障排查

### 问题 1: 无法连接 SSH

**现象**:
```
ssh: connect to host xxx port 22: Connection timed out
```

**排查步骤**:
```
1. 检查实例状态：是否是"运行中"
2. 检查公网 IP: 是否有公有 IP
3. 检查安全组：是否开放 22 端口
4. 检查密钥权限：chmod 400 key.pem
5. 检查用户名：是否用了正确的用户名
```

### 问题 2: 权限被拒绝

**现象**:
```
Permission denied (publickey)
```

**解决方案**:
```
1. 确认使用了正确的密钥
2. 检查密钥文件权限
3. 确认用户名正确
4. 尝试重新创建密钥对
```

### 问题 3: 网站无法访问

**现象**:
```
浏览器显示：无法访问此网站
```

**排查**:
```
1. 检查 Apache 是否运行：systemctl status httpd
2. 检查安全组 80 端口
3. 检查防火墙：sudo firewall-cmd --list-all
4. 检查监听端口：sudo netstat -tulpn | grep :80
```

### 问题 4: 磁盘空间不足

**现象**:
```
No space left on device
```

**解决**:
```bash
# 查看磁盘使用
df -h

# 查找大文件
sudo du -ah / | sort -rh | head -20

# 清理空间
sudo yum clean all
rm -rf ~/downloads/*
```

### 问题 5: 内存不足

**现象**:
```
Out of memory
进程被 kill
```

**解决**:
```bash
# 查看内存使用
free -h
top

# 创建 swap 分区
sudo dd if=/dev/zero of=/swapfile bs=1M count=1024
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 永久生效
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

## 3.14 常见问题 FAQ

### Q1: 实例启动后多久可以使用？
**A**: 通常 30 秒到 2 分钟。首次启动可能需要更长时间。

### Q2: 公网 IP 会变吗？
**A**: 会。停止再启动后公网 IP 会改变。如需固定 IP，使用 Elastic IP。

### Q3: 免费套餐的 750 小时怎么算？
**A**: 一个月最多 744 小时（24×31）。可以运行 1 台实例整月，或多台累计 750 小时。

### Q4: 如何备份 EC2?
**A**: 创建 AMI 镜像或 EBS 快照。建议定期备份重要数据。

### Q5: 实例变慢怎么办？
**A**: 
1. 检查 CPU 和内存使用率
2. 查看磁盘空间
3. 检查网络连接数
4. 考虑升级实例类型

### Q6: 可以同时运行多台实例吗？
**A**: 可以，但注意总费用。新账户可能有服务限额。

### Q7: 如何彻底删除 EC2?
**A**: 
1. 停止实例
2. 终止实例
3. 删除相关 EBS 卷
4. 删除安全组
5. 释放 Elastic IP
