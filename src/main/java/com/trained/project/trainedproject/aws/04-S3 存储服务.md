## 4.1 S3 基础概念

### 4.1.1 什么是 S3？

**S3 = Simple Storage Service（简单存储服务）**

简单理解：
```
S3 = 云端的超大 U 盘

特点:
  ✓ 存储任意类型的文件
  ✓ 容量无限扩展
  ✓ 按实际使用量付费
  ✓ 99.999999999% 数据持久性
```

### 4.1.2 核心术语

#### 存储桶 (Bucket)
```
Bucket = 文件夹容器
类似：Windows 中的磁盘分区

命名规则:
  - 全局唯一（全世界不能重名）
  - 长度 3-63 个字符
  - 只能包含小写字母、数字、连字符
  - 必须以字母或数字开头
  
例子:
  ✅ my-bucket-2024
  ✅ falak-personal-files
  ❌ My-Bucket (有大写)
  ❌ my_bucket (有下划线)
```

#### 对象 (Object)
```
Object = 存储在 S3 中的文件
包括：文件本身 + 元数据

组成:
  - Key (键): 文件的完整路径
  - Value (值): 文件的实际内容
  - Metadata (元数据): 描述信息
  - Version ID (版本 ID): 版本标识
```

#### 键 (Key)
```
Key = 对象的完整路径

格式:
  bucket-name/folder/subfolder/file.txt
  
例子:
  photos/2024/vacation/beach.jpg
  documents/report.pdf
  backups/database/2024-01-15.sql
```

### 4.1.3 使用场景

**适合用 S3 的情况**:
```
✅ 存储图片和视频
✅ 备份和归档
✅ 静态网站托管
✅ 大数据分析数据源
✅ 软件分发
✅ 日志存储
✅ 用户上传的内容
```

**不适合用 S3 的情况**:
```
❌ 频繁修改的文件（用 EBS）
❌ 需要低延迟的数据库（用 RDS/DynamoDB）
❌ 操作系统启动盘（用 EBS）
```

## 4.2 创建第一个存储桶

### 4.2.1 详细步骤

#### 步骤 1: 进入 S3 控制台
```
1. 登录 AWS 控制台
2. 搜索 "S3"
3. 点击 "S3" 服务
```

#### 步骤 2: 创建存储桶
```
1. 点击橙色按钮 "创建存储桶"
2. 填写存储桶名称
```

**命名示例**:
```
✅ falak-test-bucket-2024
✅ my-first-s3-bucket
✅ personal-files-storage
❌ test (太短)
❌ MYBUCKET (大写)
```

#### 步骤 3: 选择区域
```
区域：选择离你最近的
推荐：
  - 新加坡 (ap-southeast-1)
  - 东京 (ap-northeast-1)
  
注意：
  ✓ 区域影响访问速度
  ✓ 不同区域价格可能不同
  ✓ 创建后无法更改区域
```

#### 步骤 4: 配置选项

**A. 阻止公共访问设置**
```
☑ 阻止所有公共访问（推荐新手）

说明:
  ✓ 防止意外公开数据
  ✓ 后续可以按需开放
  ✓ 安全优先
```

**B. 存储桶版本控制**
```
○ 启用（推荐）
  - 保留所有版本的对象
  - 可以恢复误删的文件
  - 会产生额外存储费
  
○ 暂停（默认）
  - 只保留当前版本
  - 删除无法恢复
```

建议：学习期间先暂停，熟悉后再启用

**C. 默认加密**
```
加密类型:
  ○ 无（默认）
  ● SSE-S3 (服务器端加密)
    - AES-256 加密
    - AWS 管理密钥
    - 免费
    
  ● SSE-KMS
    - KMS 管理密钥
    - 有审计功能
    - 额外费用
```

建议：选择 SSE-S3（免费且安全）

**D. 对象锁定**
```
用于合规性要求
一般用途不需要
保持禁用即可
```

#### 步骤 5: 完成创建
```
滚动到底部
点击 "创建存储桶"
看到成功消息
```

## 4.3 上传和管理对象

### 4.3.1 上传文件（控制台方式）

#### 步骤 1: 打开存储桶
```
1. S3 控制台
2. 点击存储桶名称
3. 进入存储桶详情
```

#### 步骤 2: 上传文件
```
1. 点击 "上传" 按钮
2. 点击 "添加文件"
3. 选择本地文件
4. 或直接拖拽文件到浏览器
```

#### 步骤 3: 设置权限（可选）
```
如果启用了阻止公共访问：
  - 文件默认私有
  - 只有所有者能访问
  
如果需要公开：
  - 取消阻止公共访问
  - 或预签名 URL
```

#### 步骤 4: 设置元数据（可选）
```
可以设置:
  - Content-Type: 文件类型
  - Cache-Control: 缓存控制
  - Custom Metadata: 自定义元数据
```

#### 步骤 5: 完成上传
```
点击 "上传" 按钮
等待进度条完成
查看上传的文件
```

### 4.3.2 上传文件（CLI 方式）

```bash
# 上传单个文件
aws s3 cp myfile.txt s3://bucket-name/

# 上传到子目录
aws s3 cp myfile.txt s3://bucket-name/folder/

# 上传整个目录
aws s3 cp ./local-folder/ s3://bucket-name/ --recursive

# 上传并设置存储类别
aws s3 cp largefile.zip s3://bucket-name/ \
  --storage-class STANDARD_IA
```

### 4.3.3 下载文件

```bash
# 下载单个文件
aws s3 cp s3://bucket-name/file.txt .

# 下载到指定位置
aws s3 cp s3://bucket-name/file.txt ~/Downloads/

# 下载整个目录
aws s3 cp s3://bucket-name/folder/ ./local-folder/ --recursive
```

### 4.3.4 同步文件

```bash
# 本地同步到 S3
aws s3 sync ./local-folder s3://bucket-name/

# S3 同步到本地
aws s3 sync s3://bucket-name/ ./local-backup/

# 两个 S3 之间同步
aws s3 sync s3://bucket1/ s3://bucket2/
```

### 4.3.5 删除对象

```bash
# 删除单个对象
aws s3 rm s3://bucket-name/file.txt

# 删除多个对象
aws s3 rm s3://bucket-name/ --recursive

# 删除空存储桶
aws s3 rb s3://bucket-name
```

## 4.4 存储类别

### 4.4.1 标准存储类对比

| 存储类别 | 适用场景 | 存储价格 | 取回费用 | 最低存储期限 |
|----------|----------|----------|----------|--------------|
| **S3 Standard** | 频繁访问 | 高 | 无 | 无 |
| **S3 Intelligent-Tiering** | 访问模式变化 | 中等 | 无 | 30 天 |
| **S3 Standard-IA** | 不常访问 | 中低 | 有 | 30 天 |
| **S3 One Zone-IA** | 不常访问，可重建 | 低 | 有 | 30 天 |
| **S3 Glacier Instant Retrieval** | 季度访问 | 很低 | 有 | 90 天 |
| **S3 Glacier Flexible Rate** | 月度访问 | 极低 | 有 | 90 天 |
| **S3 Glacier Deep Archive** | 年度访问 | 最低 | 有 | 180 天 |

### 4.4.2 各存储类详解

#### S3 Standard（标准）
```
特点:
  ✓ 高耐用性和可用性
  ✓ 毫秒级访问
  ✓ 无最低存储期限
  
适用:
  - 网站图片
  - 移动应用内容
  - 频繁访问的数据
  
价格示例:
  $0.023/GB/月
```

#### S3 Intelligent-Tiering（智能分层）
```
特点:
  ✓ 自动移动到最优存储层
  ✓ 无需取回费用
  ✓ 基于访问模式
  
层级:
  - 频繁访问层
  - 不频繁访问层（30 天未访问）
  - 归档即时访问层（90 天未访问）
  
适用:
  - 访问模式不确定的数据
  - 长期增长的数据集
```

#### S3 Standard-IA（不频繁访问）
```
特点:
  ✓ 同 Standard 性能
  ✓ 存储费更低
  ✓ 有取回费用
  
适用:
  - 备份
  - 灾难恢复数据
  - 每月访问几次的数据
  
价格:
  存储：$0.0125/GB/月
  取回：$0.01/GB
```

#### S3 Glacier 系列（归档）

**Glacier Instant Retrieval（即时取回）**:
```
取回时间：毫秒级
适用：季度访问的数据
最低期限：90 天
```

**Glacier Flexible Rate（灵活费率）**:
```
取回时间：分钟到小时
适用：月度访问的数据
最低期限：90 天
```

**Glacier Deep Archive（深度归档）**:
```
取回时间：小时级
适用：合规归档
最低期限：180 天
最便宜的选择
```

### 4.4.3 更改存储类别

**控制台方式**:
```
1. 选择对象
2. 点击 "操作"
3. 选择 "更改存储类别"
4. 选择新类别
5. 保存
```

**CLI 方式**:
```bash
# 复制对象并更改存储类别
aws s3 cp s3://bucket/file.txt s3://bucket/file.txt \
```

## 4.5 权限管理

### 4.5.1 权限类型

#### 1. IAM 策略
```
控制用户/角色对 S3 的访问
附加到 IAM 用户或角色
```

**示例策略**:
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject",
        "s3:DeleteObject"
      ],
      "Resource": "arn:aws:s3:::my-bucket/*"
    }
  ]
}
```

#### 2. 存储桶策略
```
附加到存储桶
控制所有访问
基于资源的策略
```

**公开读取策略**:
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": ["s3:GetObject"],
      "Resource": "arn:aws:s3:::my-bucket/*"
    }
  ]
}
```

#### 3. ACL（访问控制列表）
```
较老的权限系统
仍在使用
推荐使用策略代替
```

### 4.5.2 生成预签名 URL

**什么是预签名 URL？**
```
临时访问链接
包含认证信息
有过期时间
无需公开存储桶
```

**使用场景**:
```
✓ 允许用户临时下载私有文件
✓ 上传文件到指定位置
✓ 分享给特定人员
```

**生成方法（CLI）**:
```bash
# 生成下载 URL（有效期 1 小时）
aws s3 presign s3://bucket-name/file.txt --expires-in 3600

# 生成上传 URL
aws s3 presign s3://bucket-name/upload-folder/file.txt \
  --expires-in 1800
```

**输出示例**:
```
https://bucket-name.s3.amazonaws.com/file.txt?
X-Amz-Algorithm=AWS4-HMAC-SHA256&
X-Amz-Credential=AKIAIOSFODNN7EXAMPLE&
X-Amz-Date=20240115T120000Z&
X-Amz-Expires=3600&
X-Amz-Signature=abcdef123456...&
X-Amz-SignedHeaders=host
```

## 4.6 版本控制

### 4.6.1 什么是版本控制？

```
版本控制 = 保存对象的所有版本

好处:
  ✓ 恢复误删的文件
  ✓ 回溯到历史版本
  ✓ 防止意外覆盖
```

### 4.6.2 启用版本控制

**控制台方式**:
```
1. 进入存储桶
2. 属性标签
3. 找到"存储桶版本控制"
4. 点击"编辑"
5. 选择"启用"
6. 保存
```

**CLI 方式**:
```bash
aws s3api put-bucket-versioning \
  --bucket my-bucket \
  --versioning-configuration Status=Enabled
```

### 4.6.3 版本状态

| 状态 | 说明 |
|------|------|
| 未启用 | 无版本控制（默认） |
| 已启用 | 版本控制开启 |
| 已暂停 | 曾启用过，现暂停 |

### 4.6.4 管理版本

**查看对象版本**:
```
1. 进入存储桶
2. 点击对象
3. 查看"版本"标签
4. 看到所有版本
```

**恢复旧版本**:
```
1. 找到要恢复的版本
2. 点击"操作"
3. 选择"还原到此版本"
4. 确认
```

**删除特定版本**:
```
1. 显示对象版本
2. 选择要删除的版本
3. 点击"删除"
4. 输入对象名称确认
```

## 4.7 生命周期管理

### 4.7.1 什么是生命周期规则？

```
自动转换存储类别
自动过期对象
基于规则自动化
节省存储成本
```

### 4.7.2 常见规则示例

#### 规则 1: 日志文件自动归档
```
前缀：logs/
动作:
  - 30 天后 → Standard-IA
  - 90 天后 → Glacier Instant Retrieval
  - 365 天后 → 过期删除
```

#### 规则 2: 上传文件智能分层
```
前缀：uploads/
动作:
  - 立即 → Intelligent-Tiering
  - 自动优化存储成本
```

#### 规则 3: 临时文件自动清理
```
前缀：temp/
动作:
  - 7 天后 → 过期删除
```

#### 规则 4: 旧版本自动清理
```
应用于：所有对象
当前版本:
  - 365 天后 → Glacier
  - 730 天后 → 删除
  
旧版本:
  - 30 天后 → Glacier
  - 90 天后 → 删除
```

### 4.7.3 创建生命周期规则

**详细步骤**:

```
步骤 1: 进入存储桶

步骤 2: 管理标签

步骤 3: 生命周期规则

步骤 4: 创建规则

步骤 5: 配置规则
  规则名称：MoveToGlacier
  前缀：documents/
  
步骤 6: 添加转换
  当前版本:
    - 30 天后 → Standard-IA
    - 90 天后 → Glacier Instant
    
  旧版本:
    - 30 天后 → Glacier
    - 90 天后 → 删除
    
步骤 7: 审核并创建
```

## 4.8 静态网站托管

### 4.8.1 什么是 S3 静态网站？

```
托管纯静态网站
HTML + CSS + JavaScript
无需服务器
超便宜
```

**适用**:
```
✅ 个人博客
✅ 作品集网站
✅ 产品页面
✅ 文档站点
```

**不适用**:
```
❌ 需要后端处理（PHP/Python）
❌ 数据库交互
❌ 用户登录系统
```

### 4.8.2 创建静态网站（详细步骤）

#### 步骤 1: 创建存储桶
```
名称：必须与域名一致
例如：www.example.com

区域：选择近的
```

```
⚠️ 重要：必须取消阻止！

1. 编辑阻止公共访问设置
2. 取消勾选"阻止所有公共访问"
3. 确认此存储桶将公开
```

```
1. 属性标签
2. 找到"静态网站托管"
3. 点击"编辑"
4. 启用
```

**配置**:
```
索引文档：index.html
```

#### 步骤 4: 添加存储桶策略

## 4.9 数据传输和成本

### 4.9.1 定价组成

```
总费用 = 
  存储费用 + 
  请求费用 + 
  数据传输费用 + 
  管理功能费用
```

### 4.9.2 存储费用示例

假设使用 Standard 存储（$0.023/GB/月）:

```
1 GB:   $0.023/月
10 GB:  $0.23/月
100 GB: $2.30/月
1 TB:   $23.00/月
```

### 4.9.3 请求费用

```
PUT/COPY/POST/LIST:
  $0.005 / 1000 次请求
  
GET/SELECT:
  $0.0004 / 1000 次请求
```

### 4.9.4 数据传输费用

**流入 S3（免费）**:
```
从互联网传入：免费
从其他 AWS 服务：通常免费
```

**流出 S3（收费）**:
```
到互联网:
```

## 4.10 数据保护

### 4.10.1 加密选项

#### 服务端加密 (SSE)

**SSE-S3**:
```
AES-256 加密
AWS 管理密钥
免费
推荐用于一般数据
```

**SSE-KMS**:
```
KMS 管理密钥
有审计追踪
可控制密钥访问
额外费用
```

**SSE-C**:
```
客户提供密钥
AWS 执行加密
自己管理密钥
风险较高
```

#### 客户端加密
```
上传前本地加密
完全控制密钥
增加复杂性
```

### 4.10.2 启用加密

**创建时启用**:
```
创建存储桶时
选择"默认加密"
选择 SSE-S3
```

**现有存储桶**:
```
1. 属性 → 默认加密
2. 编辑
3. 启用
4. 选择加密类型
```

### 4.10.3 防止意外删除

**启用版本控制**:
```
保留所有版本
轻松恢复
```

**MFA Delete**:
```
删除需要 MFA
更高安全性
```

**对象锁定**:
```
WORM（一次写入多次读取）
合规性要求
防止删除
```

## 4.13 故障排查

### 问题 1: Access Denied

**原因**:
```

- 存储桶策略限制
- 公共访问被阻止

```

**解决**:

**解决**:

## 4.16 常见问题 FAQ

### Q1: S3 最大存储容量是多少？
**A**: 理论上无限。单个对象最大 5TB。

### Q2: 存储桶创建后能改名吗？
**A**: 不能。需创建新桶并迁移数据。

### Q3: 如何批量删除对象？
**A**: 使用 `aws s3 rm s3://bucket/ --recursive`

### Q4: 免费套餐有多少额度？
**A**: 
- Standard 存储：5GB
- 每月 20,000 次 GET 请求
- 每月 2,000 次 PUT 请求

### Q5: 如何加速 S3 访问？
**A**: 
1. 选择近的区域
2. 使用 CloudFront CDN
3. 启用 S3 Transfer Acceleration

### Q6: 可以挂载 S3 为本地磁盘吗？
**A**: 可以，使用第三方工具如 s3fs、Mountain Duck。

### Q7: 如何迁移其他云存储到 S3？
**A**: 使用 AWS DataSync、Snowball 或第三方工具。
